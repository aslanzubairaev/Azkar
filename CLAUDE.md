## 1. Комментарии (АБСОЛЮТНЫЙ ПРИОРИТЕТ)

1. **Каждый файл `.tsx` начинается с заголовочного комментария в строке 0**, который простым языком объясняет:
   - для чего нужен этот файл,
   - что он показывает на экране,
   - что человек может с этим сделать.

   Пример:
   ```tsx
   /*
    Этот файл определяет страницу профиля.
    Он показывает данные вошедшего пользователя (имя, фото, email).
    Он также позволяет менять некоторые простые детали, например город или имя.
   */
   ```

2. **У каждого важного элемента должен быть комментарий выше:**
   - элемента, который хранит информацию для страницы,
   - элемента, который срабатывает автоматически при загрузке,
   - функций, которые запускаются по клику или отправке формы,
   - крупных частей интерфейса (список, форма, модальное окно и т. д.).

   Комментарии пишутся простым языком, без жаргона (без «hook», «state», «props» и т. п.).
   Если читать только комментарии в файле, человек без опыта в коде должен понять, что делает страница.

3. **Комментарии всегда актуальны** — если поведение меняется, комментарий сразу меняется.

## 2. Структура файлов и папка `components`

1. В проекте **всегда есть папка `components`**.
2. **Все переиспользуемые компоненты** создаются в `components/`.
3. Файлы страниц (в `app/`) должны оставаться максимально простыми — только используют компоненты, не дублируют интерфейс.

## 3. Стили (CSS)

1. Каждый компонент имеет свой файл стилей **`ComponentName.module.css`** рядом с файлом компонента.
2. Каждая страница имеет свой файл стилей **`page.module.css`** рядом с `page.tsx` в той же папке.
3. **`globals.css`** — только переменные, сброс стилей и шрифты. Никаких стилей конкретных страниц или компонентов.
4. Никаких инлайн-стилей (`style={{}}`), кроме динамических значений (например, ширина прогресс-бара в процентах).

## 4. Запрещено

1. **Никаких HOC** — не создавать компонент, который принимает компонент и возвращает новый.
2. **Никаких render props** — не передавать функцию для отрисовки как основную prop.
3. **Никаких кастомных хуков** — не создавать функции, начинающиеся с `use`. Вся логика остаётся внутри самих компонентов.
